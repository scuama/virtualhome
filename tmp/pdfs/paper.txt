                            Abstract
  Continual embodied instruction following (CEIF) is preva-
  lent in long-term human-agent cohabitation tasks such as
  home service, elderly care, and autonomous inspection. CEIF
  is particularly challenging in open dynamic environments
  which is interved by human activity between consecutive ob-
  servations, invalidating the agent’s previous recognition of
  environment state. Existing methods make decisions primar-
  ily from surface-level information of the environment, which
  causes planning to depend on stale or misaligned information
  and involve unavailable objects, ultimately leading to failures.
  We propose RoboState, a state-aware framework for CEIF in
  open dynamic environments, which operates through a three-
  stage pipeline of (1) parsing an instruction into goal states,     Figure 1: Continual Embodied instruction following in open

4.4     Robustness and Cost
Backbone Robustness. We further evaluate RoboState
with multiple LLM backbones under the default setting,
which cover both open and closed source models from dif-
ferent types and scales.
   As shown in Table 4, RoboState maintains high SR and PS
regardless of model types and scales. Specifically, SR ranges
from xx% to xx%, and PS ranges from xx to xx timesteps.
In particular, RoboState continues to perform well on the
locally-deployed 8B-parameter Llama-3 model. The reason
is that, RoboState uses the LLM for semantic interpreta-
tion within each module, whereas module outputs are con-
strained by explicit structures. These constraints reduce the
dependency on unconstrained LLM generation and limit its
sensitivity to the type and scale of the backbones.
Clarification Cost. We conduct this experiment on the
parameter-missing subset, which is the only scenario subset
designed to require clarification. We vary the budget b over
0, 1, 2, 3, where b specifies the maximum number of ques-
tions permitted during each execution. Section ?? both use
b = 0. We measure the clarification effectiveness by report-
ing SR and ClarCost. ClarCost is upper-bounded by b but
does not necessarily attain this bound. During evaluation, a
simulated user answers clarification queries using the anno-
tated ground-truth parameters.

and largely saturates once b reaches 2, which indicate that
user clarification therefore provides an effective optional ex-
