# 场景配置文件支持能力手册

为了支持大模型在 VirtualHome 中的消融实验，我们的场景配置文件（JSON 格式）提供了一套强大的环境“上帝之手”功能。在构造数据集时，你们可以通过配置以下字段，随意定制物理环境和突发事件。

## 1. 基础任务设定

*   **任务指令 (`instruction`)**：直接给到机器人的自然语言指令。例如 `"去冰箱拿个苹果"`，或者模糊指令 `"去拿个水果来"`。
*   **全局规则 (`global_rules`)**：赋予机器人必须遵守的长期规则。例如 `["NO_WIPE_WHILE_ON"]` 表示“电视机通电时禁止擦拭”。如果机器人违规，评测系统会直接拦截并判败。
*   **切换场景 (`environment_id`)**：指定出生在 0~6 号不同的 VirtualHome 居家户型中。

## 2. 初始环境篡改 (`setup`)

在任务开始的第 0 步，你可以利用 `setup` 字段对环境进行任意的静态篡改：

*   **锁定出生点 (`init_room`)**：强制机器人出生在指定的房间（例如 `"kitchen"`），避免开局迷路导致测试失败。
*   **移除物品 (`remove_nodes`)**：输入物品名称的列表（如 `["apple", "milk"]`），将其从世界上彻底抹除。机器人无论怎么找都找不到。
*   **修改物品属性 (`modify_properties`)**：剥夺或赋予物品的物理交互能力。
    *   *剥夺属性*：例如将 `fridge` 的 `CAN_OPEN` 属性剥夺，制造出“打不开的冰箱”这种物理死局。
    *   *赋予属性*：例如将原本不能拿起的物品添加 `GRABBABLE` 属性。
*   **修改物品状态 (`modify_states`)**：直接设定物品的初始状态。例如将微波炉强制设置为 `["DIRTY", "PLUGGED_OUT", "OFF"]`（脏、断电、关机），用来测试机器人是否能理清开机前的依赖顺序。
*   **凭空添加物品 (`add_nodes`)**：在指定位置凭空生成新的物品，支持自定义状态。例如：在桌子上同时生成两个 `mug`，一个状态是 `CLEAN`，另一个是 `DIRTY`（用于测试机器人能否区分同类物品）。
*   **强占机器人双手 (`agent_inventory`)**：开局强制在机器人的左右手里塞入指定物品。例如 `["apple", "rag"]`，用来制造资源竞争——机器人在想拿新东西前，必须先规划出“放下手中物品”的步骤。

## 3. 实时动态事件 (`dynamic_events`)

用来测试机器人在执行任务途中的实时反应能力。系统会监控机器人与目标物品的距离，一旦靠近就会触发事件：

*   **物品瞬移 (`teleport`)**：当机器人靠近目标物品（如苹果）时，苹果瞬间被传送到了另一个房间的地板上。用来测试机器人是否会因为记忆错乱而产生幻觉，或者能否重新规划路线。
*   **凭空消失 (`vanish_reappear`)**：当机器人靠近物品时，物品瞬间消失。用来模拟被其他人拿走的情况。

---
**配置提示**：所有的篡改手段都可以任意组合，比如你可以让微波炉既**打不开**、又**断着电**，甚至在机器人走近时发生**瞬移**！
