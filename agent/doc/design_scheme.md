# 智能体消融实验数据集设计方案 (定义完全修正版)

## 1. Goal Reasoning (G) - 意图提取与目标状态维护

*   **[G1] 正常条件触发 (Standard Trigger)**
    *   **理解**：基本的条件-动作判断。判断当前环境是否满足触发条件，并生成正确的期望状态。这是最基本的测试基线，无需特殊底层节点构造。

*   **[G2] 意图模糊/需问答补充 (Ambiguous Intent)**
    *   **理解**：指令缺少关键参数，测试模型是直接产生幻觉（盲目操作）还是能挂起任务主动提问。
    *   **构造方法**：用户指令本身极度模糊。例如“去拿个水果来”（目标类目模糊，不知道要拿哪个具体水果），或者“把杯子拿过去”（目标地点完全缺失）。这种模糊指令导致机器人无法确定目标物品或地点，促使其主动发起问答补充信息。

*   **[G3] 条件恒不满足 (Impossible Condition)**
    *   **理解**：专指“如果A，则去做B，但是A这个前提永远不满足”。通过剥夺底层物理属性使前置条件永久失效。
    *   **构造方法**：指令“如果某个容器是打开的，就把某个东西放进去”。但我们在初始化时剥夺该容器的底层开关功能，导致“门是开着的”这个前置条件永远不可能满足。

*   **[G4] 路径不可达/手段缺失 (Path Unreachable / Missing Method)**
    *   **理解**：意图清晰，但完成任务的常规手段或路径被阻断（例如工具锁死），需要智能体寻找替代方案。
    *   **构造方法**：指令“用微波炉去热牛奶”，但我们在初始化时将微波炉锁死（例如剥夺打开或加热属性）。智能体需要发现微波炉无法使用，并在环境中寻找其他替代方案（例如用炉灶加热）。

## 2. State-Consistent Memory (M) - 状态一致性记忆

*   **[M1] 物品短暂消失后重现/部分遮挡 (Disappearance & Occlusion)**
    *   **理解**：单纯测试追踪丢失视线物品的能力。剥除任何容器遮挡，纯粹模拟被人拿走又放回来的现象。
    *   **构造方法**：在智能体前往拿水杯（`mug`）的途中，脚本强行删除水杯，2-3 步之后在原位重新生成放回来。

*   **[M2] 多个相似物体共存/实例混淆 (Instance Confusion)**
    *   **理解**：场景中有多个同类物品，必须通过记忆它们的不同状态/历史操作来区分。
    *   **构造方法 A (状态区分)**：生成两个完全同类的 `mug`，一个状态是 `['DIRTY']`，另一个是 `['CLEAN']`。指令要求“洗脏杯子”。
    *   **构造方法 B (空间位置区分)**：沙发上和桌子上各有一本书（`book`），智能体需准确记住“沙发上的那本”的特定 ID 并操作。

*   **[M3] 动态全局规则变化 (Dynamic Global Rules)**
    *   **理解**：模拟用户偏好或环境规则的动态变化，测试智能体是否能实时遵守新设定的规则。
    *   **构造方法**：指令设定临时全局规则（例如“当前时间段内，禁止进入卧室”）。后续给出指令“去拿某物”，而该物品在卧室中。智能体遇到规则冲突时，可以选择执行等待（Wait）操作，或者中止并汇报冲突，直到限制解除。

*   **[M4] 植入错误记忆 / 幻觉诱导 (Implanted Memory)**
    *   **理解**：指令中包含与环境事实不符的错误前置信息（植入记忆），测试智能体是否会被误导产生幻觉，还是能通过观察纠正记忆。
    *   **构造方法**：指令中强行给机器人植入一段记忆：“苹果刚才放在桌子上，你去拿过来”。但实际上苹果已经被拿走不在桌子上了。智能体到达桌子后发现不符，需要依靠自身的探索能力重新去其他地方（如冰箱）寻找苹果。

## 3. SDG Planning (P) - 目标驱动的任务规划

*   **[P1] 资源竞争/依赖冲突 (Resource Competition)**
    *   **理解**：单个任务拆分出的多个子任务需要使用同一个物理资源（如微波炉），从而产生资源竞争，需要规划合理的先后顺序。
    *   **构造方法**：指令要求“加热两份不同的食物（例如牛奶和汉堡）”。微波炉容量有限（或者物理常识不允许同时加热），智能体必须规划出串行处理的步骤，而不能试图同时塞入加热。

*   **[P2] 动作合并/并行优化 (Action Merging / Optimization)**
    *   **理解**：利用双手资源，当在同一处有多个小物品需要搬运时，规划“一次性拿走两个”以节省步数。
    *   **构造方法 A (双物同源)**：把桌上的两个苹果拿到厨房。最优规划为一手抓一个，一次性运走。
    *   **构造方法 B (中转并单)**：一手拿盘子，一手拿杯子，顺路一起带到水槽清洗。

*   **[P3] 严格复杂时序依赖 (Complex Temporal Dependency)**
    *   **理解**：涉及多个物品之间相互依赖的状态变化链路，某一步骤必须依赖其他物品特定状态的达成。
    *   **构造方法**：指令“加热牛奶”。初始状态下微波炉的插头是被拔掉的。这个任务涉及到牛奶和微波炉两个物品的状态变化：牛奶的加热必须依赖于微波炉处于“已通电（PLUGGED_IN）”且“已开启（ON）”的状态。智能体必须先理清并执行微波炉的状态转移，才能完成牛奶的加热。
